## 0. 一句话定义
`MUSE AI Studio` 是面向服装与时尚行业的双端 Web 应用：
- `Studio Dashboard`：给设计师/买手/运营使用的“创作工作台”，用于选款、搭配、AI 生成图/视频、管理内容产出。
- `Pick Board`：给达人/消费者使用的“沉浸式内容场”，用于浏览、互动、种草、反馈偏好。

核心目标是打通“素材 -> 搭配 -> 生成 -> 发布 -> 互动反馈 -> 反哺下一轮创作”的闭环。

## 1. 背景与问题

### 1.1 现状痛点
- 服装内容生产链路割裂：灵感、选款、搭配、拍摄、发布、反馈分散在多个工具中。
- 高质量内容成本高、周期长：拍摄组织重、复用低、试错慢。
- 消费端反馈回流慢：达人/消费者偏好无法快速反哺到创作端。

### 1.2 机会点
- 通过 AI 把“搭配 + 视觉生成 + 反馈分析”串成统一工作流。
- 用 Demo 证明：少人团队借助 AI coding 也能快速产出可演示、可迭代的时尚内容系统。

## 2. 产品目标（Demo 阶段）

## 2.1 业务目标
- 在 `1 周` 内完成可演示的端到端产品闭环。
- 让品牌内容团队看到“更快出片/出内容”的效率提升。
- 让达人/消费者端产生真实互动行为，为下一轮推荐与搭配提供信号。

## 2.2 产品目标（可验证）
- 用户可从本地导入图片到 `Studio Dashboard` 中，完成选品和搭配
- 每套搭配可触发 AI 虚拟拍摄（图像为主，视频可为简化版）并支持反复修改，最终采纳进入结果流。
- 生成内容可发布到 `Pick Board`，并支持点赞/收藏/评论/选款等互动。
- Studio 端可看到基础反馈指标（互动量、收藏率、热门风格标签）。

## 2.3 非目标（本期不做）
- 不做完整电商交易链路（下单/支付/履约）。
- 不做复杂组织权限体系（先用简化角色）。
- 不做大规模高并发和全球化架构（以 Demo 稳定性为先）。

## 4. 产品范围（模块定义）

## 4.1 模块 A：Studio Dashboard（时尚版 Figma）
Studio 端核心形态是“画布工作台“，即” 左侧素材流 + 中部画布操作区 + 右侧生成结果列表”。

负责人：@gaoran、@xuxianghao、sichaung

### P0（Demo 必做）
1. home：
	1. 展示数据指标（创建内容数量、点赞数、……）
	2. 展示方案 list（以设计方案为单位，每个设计方案中可以包含很多套搭配、很多套内容）
2. `素材池`：上传/导入素材，支持批量多图片导入，支持识别基础标签（品类、颜色、季节、风格）。
3. `搭配画布`：拖拽选品组成 Look（2~6 件单品）。
4. `AI 搭配建议`：对选品生成搭配卡片（含文案描述）。*
5. `拍摄触发`：点击“拍摄”后展开生成面板
	1. 可控项：指定模特、指定场景、指定姿势（可选）、其他自定义指令
	2. 生成任务（状态：排队/处理中/完成/失败）。
	3. 结果展示：输出在画布中，被采纳的结果会放入右侧列表
6. `结果面板`：展示采纳结果（图片必做，视频可先占位/可选）。
7. `发布入按钮：可将结果发送到 Pick Board。
8. 设置：支持查看和修改调用gemini 模型时的模型型号、系统提示词；支持查看版本号


## 4.2 模块 B：Pick Board（时尚版 INS/小红书）

负责人：@zhangmuhuo、@gaotianjiao

### P0（Demo 必做）
1. `沉浸式 Feed`：瀑布流/卡片流展示 Look 内容。
2. `内容详情`：可查看内容、查看搭配单品和描述文案。
3. `互动行为`：点赞、收藏、发起 tryon、选品、评论（可先做简化交互）。
4. `回流信号`：互动数据写回 Studio 指标面板。

### P1（可选增强）
1. “一键同风格”二次生成入口。
2. 达人专题板（按风格主题聚合内容）。

## 4.3 闭环定义（必须打通）
`Studio 生成内容 -> 发布到 Pick Board -> 用户互动 -> 反馈回 Studio（数据/标签）`

## 5. 关键用户流程

## 5.1 创作流程（设计师/买手/运营）
1. 新建设计方案，导入或选择素材。
2. 在画布中组合单品形成 Look。
3. 触发 AI 生成搭配图（可带风格提示词）。
4. 点击“拍摄”进入任务队列，产出图像（和可选视频）。
5. 选择结果并发布至 Pick Board。

## 5.2 消费流程（达人/消费者）
1. 进入 Pick Board 浏览内容流。
2. 对内容点赞/收藏/评论。
3. 打开内容详情查看单品组合与风格描述。

## 5.3 反馈回流流程（运营）
1. 查看互动数据（曝光、点赞、收藏、评论）。
2. 识别高表现风格标签。
3. 将标签反馈到下一轮 Studio 生成策略。



## 11. Demo 验收标准（Definition of Success）
- 能从素材中完成至少 `1 条完整创作链路`（选品 -> 搭配 -> 生成 -> 发布）。
- Pick Board 可看到发布内容并产生真实互动记录。
- Studio 可看到来自 Pick Board 的回流数据。
- 核心链路演示时无阻断性错误，关键页面可连续演示 `>= 15 分钟`。
